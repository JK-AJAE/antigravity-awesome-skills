# 사용 가이드: 스킬을 실제로 사용하는 방법

> **설치 후에 무엇을 해야 할지 모르겠나요?** 이 가이드는 여러분이 다음에 해야 할 일을 단계별로 정확하게 안내합니다.

---

## "방금 저장소를 설치했습니다. 이제 무엇을 하나요?"

훌륭한 질문입니다! 설치 후에 일어난 일과 다음 단계를 설명해 드립니다.

### 설치 후 일어난 일

`npx antigravity-awesome-skills`를 실행하거나 저장소를 복제했을 때:

✅ **1,445개 이상의 스킬 파일**이 여러분의 컴퓨터에 다운로드되었습니다. (기본 경로: `~/.gemini/antigravity/skills/`)
✅ **AI 어시스턴트가 사용할 수 있는 상태**가 되었습니다.
❌ **하지만 모든 스킬이 자동으로 활성화되지는 않았습니다.** (스킬들은 여러분의 명령을 기다리며 대기 중입니다.)

도구 상자를 설치한 것과 같다고 생각하세요. 모든 도구는 준비되었지만, 작업에 따라 **어떤 도구를 꺼내 쓸지는 여러분이 결정**해야 합니다.

---

## 1단계: "번들(Bundles)" 이해하기

**흔한 오해:** "스킬을 하나씩 따로 다운로드해야 하나요?"

**정답: 아니요!** 모든 스킬은 이미 설치되어 있습니다. 번들은 단지 역할별로 스킬들을 **분류해 놓은 가이드**일 뿐입니다.

- **전체 라이브러리 설치** = 모든 도구가 들어있는 거대한 도구 상자.
- **번들** = "목수라면 이 10가지 도구부터 써보세요"라고 표시된 **정리함**.

---

## 2단계: 스킬 실행/사용 방법

가장 중요한 부분입니다! 스킬을 사용하는 방법은 매우 간단합니다.

### 간단한 방법

**AI 어시스턴트와의 대화 중에 스킬 이름을 언급하기만 하면 됩니다.**

### 도구별 호출 구문

도구마다 약간씩 다르지만, 기본적으로는 비슷합니다:

#### Claude Code (CLI)
`>> Use @brainstorming to help me design a todo app`

#### Cursor (IDE)
`@brainstorming help me design a todo app`

#### Gemini CLI
`Use the brainstorming skill to help me plan my app`

#### Antigravity IDE
`Use @brainstorming 기능을 계획해 줘`

> **꿀팁:** 대부분의 최신 도구는 `@스킬이름` 구문을 사용합니다. 잘 모르겠다면 일단 `@`를 붙여보세요!

---

## 3단계: 프롬프트 작성 예시

### 예시 1: 새로운 프로젝트 시작
- **나쁜 예:** "할 일 앱 만드는 거 도와줘."
- **좋은 예:** "**@brainstorming**을 사용해서 사용자 인증과 클라우드 동기화 기능이 있는 할 일 앱 설계를 도와줘."

### 예시 2: 코드 리뷰
- **나쁜 예:** "내 코드 확인해 줘."
- **좋은 예:** "**@lint-and-validate**를 사용해서 `src/components/Button.tsx` 파일에 문제가 없는지 확인해 줘."

---

## 4단계: 첫 번째 스킬 따라하기 (실습)

지금 바로 다음 단계를 따라 해 보세요:

1. **상황:** 새로운 기능을 계획하고 싶습니다.
2. **스킬 선택:** `brainstorming` (Essentials 번들에 포함됨)
3. **AI에게 다음과 같이 입력하세요:**
   ```text
   Use @brainstorming 내 앱의 사용자 프로필 페이지 설계를 도와줘
   ```
4. **결과:**
   - AI가 브레인스토밍 스킬을 로드합니다.
   - AI가 여러분에게 구조화된 질문을 하나씩 던집니다.
   - 요구 사항과 설계를 이해하도록 안내합니다.
   - 여러분이 답변하면, AI가 완성된 설계 문서를 작성해 줍니다.

## 5단계: 추천 스타터 스킬

처음부터 1,445개를 다 보려고 하지 마세요. 다음 5가지만 먼저 써보세요:

1. **`@brainstorming`** - 구현 전 계획 수립
2. **`@lint-and-validate`** - 코드 깔끔하게 유지
3. **`@git-pushing`** - 작업 내용 안전하게 저장
4. **`@systematic-debugging`** - 버그를 빠르게 수정
5. **`@concise-planning`** - 작업 할 일 목록 정리
