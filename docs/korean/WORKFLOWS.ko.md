# Antigravity 워크플로우 (Workflows)

여러 스킬을 조합하여 마찰 없이 목표를 달성하기 위한 실행 가이드(Playbook)입니다.

## 워크플로우란 무엇인가요?

워크플로우는 하나의 구체적인 결과물을 만들기 위해 여러 스킬을 단계별로 연결하여 실행하는 가이드라인입니다.

- **번들(Bundles)**: 내 역할에 필요한 도구들이 무엇인지 알려줍니다.
- **워크플로우(Workflows)**: 실제 목표를 달성하기 위해 그 도구들을 **어떤 순서로 사용**해야 하는지 알려줍니다.

번들이 '도구 상자'라면, 워크플로우는 '작업 지침서'와 같습니다.

---

## 워크플로우 사용 방법

1. 전체 저장소를 설치합니다 (`npx antigravity-awesome-skills`).
2. 현재 목표에 맞는 워크플로우를 선택합니다.
3. 단계별 가이드에 따라 나열된 스킬들을 순서대로 호출합니다.
4. 각 단계에서 생성된 결과물(계획, 테스트, 검증 근거 등)을 다음 단계로 넘깁니다.

---

## 주요 워크플로우 예시

### 1. SaaS MVP 개발 (Ship SaaS MVP)
*아이디어만 있는 상태에서 배포까지의 전 과정을 다룹니다.*

1. **계획**: `@brainstorming`을 사용하여 기능 요구 사항 및 아키텍처 정의.
2. **설계**: `@concise-planning`으로 작업 단위 분할.
3. **구현**: `@react-patterns` 및 `@nextjs-best-practices`를 사용하여 핵심 기능 개발.
4. **스타일**: `@tailwind-patterns` 및 `@frontend-design`으로 UI 고도화.
5. **검증**: `@lint-and-validate`로 코드 품질 점검.
6. **배포**: `@vercel-deployment`를 통해 운영 환경 배포.

### 2. 보안 감사 워크플로우 (Security Audit)
*기존 코드의 보안 취약점을 체계적으로 점검합니다.*

1. **분석**: `@security-auditor`로 코드 베이스 전수 스캔.
2. **API 검사**: `@api-security-best-practices`로 엔드포인트 보안 확인.
3. **취약점 스캔**: `@vulnerability-scanner`를 사용하여 알려진 보안 결함 탐색.
4. **리포트**: 결과를 정리하여 우선순위가 지정된 수정 계획 수립.

## 팁: 워크플로우와 번들 조합하기
특정 워크플로우를 진행하면서 더 깊은 전문 지식이 필요하다면 [bundles.md](BUNDLES.ko.md)에서 관련 번들을 찾아 더 많은 스킬을 활용해 보세요.
